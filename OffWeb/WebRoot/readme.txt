1.
	  不需要拦截器的Action包：default
	  需添加用户拦截器的Action包：user
	  需要添加拦截器的Action需在Action前添加inter，如 UserInterAction
	  管理员Action包：manage
	  
2.	 前台传递的参数且不是model的属性则一"p_"开头

3.   Action 命名采用驼峰命名规则，如：preLogin

3.   hibernate in查询 占位符查询失败
	
4.	 js 函数命名规则：为了防止与其他函数重名，可加i开头，才用驼峰命名规则

5.	 css 命名采用 - ，如 login-div 。js 属性命名规则:加"_" 如：id="a_b"

6.	 类中的字段命名规则：驼峰

7.	 数据库表：全小写，若为关键字则在前面加t，表示（table)

8.	 数据库字段名称：驼峰，不需要添加类名，

9.	 Action 不需要捕获异常，异常已经由异常处理action 处理，根据系统设置发送、打印错误信息

10   以 i- 开头的css没有实际对应的css样式，仅用了控制内部css样式

11.	  删除数据将flag置为-100，-1不可用，1为可用
